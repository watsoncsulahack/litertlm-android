# 2026-06-20 PRoot ARM64 Build Attempt

## Result

The LiteRT-LM Android ARM64 build did not complete inside this phone-hosted
Ubuntu/PRoot environment.

The source checkout, Git LFS GPU prebuilts, Android NDK discovery, native `adb`,
and Bazel 7.6.1 setup all worked. Bazel reached target analysis, then failed when
the Android NDK repository rule tried to execute the NDK host compiler:

```text
execvp(toolchains/llvm/prebuilt/linux-x86_64/bin/clang, ...): No such file or directory
```

## Cause

This environment is `aarch64`, but the installed official Android NDK contains
Linux `x86_64` host tools:

```text
toolchains/llvm/prebuilt/linux-x86_64/bin/clang
```

That is fine on normal x86_64 Linux or GitHub Actions, but not runnable inside
ARM64 PRoot on the phone.

## Verified

The GPU prebuilt directory from upstream was pulled successfully:

- `libGemmaModelConstraintProvider.so`
- `libLiteRtGpuAccelerator.so`
- `libLiteRtOpenClAccelerator.so`

- `libLiteRtTopKWebGpuSampler.so`
- `libLiteRtWebGpuAccelerator.so`
- `libwebgpu_dawn.so`

## Practical Next Path

Build the binary on x86_64 Linux or GitHub Actions, then push the resulting
`litert_lm_main` plus the GPU `.so` files to the phone. Runtime testing still
